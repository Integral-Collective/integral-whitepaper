# 7.1 Collaborative Decision System (CDS)

The Collaborative Decision System (CDS) provides Integral’s governance and coordination substrate, enabling structured issue intake, deliberation, consensus formation, decision recording, and implementation dispatch across the broader system architecture.

This subsystem functions as both a protocol for collective intelligence and a recursive coordination layer connecting all other Integral subsystems.

---

## Structure

The CDS section is organized into:

* subsystem overview
* architecture and orchestration logic
* operational modules
* addendum describing high-bandwidth human deliberation

This segmentation enables targeted critique, modular revision, and future mapping into specification and implementation repositories.

## Module Ordering

The CDS modules represent a decision metabolism pipeline progressing from signal intake through deliberation, consensus formation, implementation dispatch, and recursive review.

While sequentially presented, modules remain architecturally modular and subject to independent revision.

---

## Contribution Guidance

Participants should:

1. Navigate to the relevant module file
2. Create structured Issues referencing that module
3. Provide evidence-based critique
4. Propose revisions via Pull Requests when appropriate

Module-level critique is preferred over subsystem-level critique.

---

## Architectural Role

The CDS subsystem serves as a proto-specification foundation for future development:

* specification repositories will inherit module boundaries defined here
* implementation repositories will map to protocol definitions described in module files
* governance experimentation will treat module files as decision objects within CDS workflows
